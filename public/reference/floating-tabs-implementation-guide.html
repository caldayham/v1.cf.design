1. Overview & Architecture

This implementation creates a desktop app-like container with floating navigation tabs at the bottom. It's particularly effective for mobile devices where bottom navigation is more thumb-friendly than top navigation bars.

Key Features

  • Mobile-first design - Tabs are easily reachable at the bottom of the screen
  • Desktop app aesthetic - Content is contained in a centered, bordered frame
  • Route-based active states - Active tab automatically highlighted based on current URL
  • Prefetched navigation - Instant tab switching with Next.js route prefetching
  • Hidden scrollbar - Clean UI with scrollable content area
  • Responsive sizing - Tabs resize for mobile vs desktop viewports

Architecture Diagram

RootLayout (app/layout.tsx) └── TabbedContentWrapper └── Frame (fixed positioning container) ├── <main> (scrollable content area) │ └── {children} (your page content) │ └── FloatingTabs (absolutely positioned at bottom) ├── Tab: About (/) ├── Tab: Portfolio (/portfolio) └── Tab: Get Quote (/get-quote)

3. Step 1: CSS Setup (globals.css)

.hide-scrollbar::-webkit-scrollbar {
  display: none;                /* Chrome, Safari, Opera */

4. Step 2: Frame Component

The Frame component creates a fixed-position container that gives your app a desktop app-like appearance. It centers the content and adds borders with rounded corners.

components/TabbedContentWrapper/Frame.tsx
'use client';
import React from 'react';

export default function Frame({ children }: { children: React.ReactNode }) {
  return (
    {/* Outer container: fixed, full viewport, centers content */}
    <div className="fixed inset-0 flex justify-center pointer-events-none z-100">

      {/* Inner container: the visible "app window" */}
      <div
        className="flex flex-col w-full max-w-6xl pointer-events-auto overflow-hidden"
        style={{
          margin: '8px',
          border: '1px solid var(--color-gray-900)',
          borderRadius: 'var(--radius)',
          background: 'var(--color-white)',
        }}
      >
        {children}
      </div>
    </div>
  );
}

How the Frame Works

CSS Property Purpose
fixed inset-0 Positions the container to cover the entire viewport
pointer-events-none Allows clicks to pass through the outer container
pointer-events-auto Re-enables clicks on the inner content container
max-w-6xl Limits width to 1152px for large screens
margin: 8px Creates a small gap around the frame (shows background)
z-100 Ensures the frame stays above other content

5. Step 3: FloatingTabs Component

This is the core component - the actual floating tab navigation. It uses Next.js routing to determine the active tab and prefetches routes for instant navigation.

components/TabbedContentWrapper/FloatingTabs.tsx
'use client';
import React, { useEffect } from 'react';
import Link from 'next/link';
import { usePathname, useRouter } from 'next/navigation';
import { Home, Briefcase } from 'lucide-react';

// Define your tabs here - customize routes and icons as needed
const tabs = [
  { name: 'About',     href: '/',          icon: Home       },
  { name: 'Portfolio', href: '/portfolio', icon: Briefcase  },
  { name: 'Get Quote', href: '/get-quote', icon: Briefcase  },
];

export default function FloatingTabs() {
  const pathname = usePathname();
  const router   = useRouter();

  /* Pre-fetch routes for instant tab switching */
  useEffect(() => {
    router.prefetch('/');
    router.prefetch('/portfolio');
  }, [router]);

  return (
    <div
      className="pointer-events-auto flex gap-0 z-20"
      style={{
        // Position at bottom center of the Frame
        position: 'absolute',
        left: '50%',
        transform: 'translateX(-50%)',
        bottom: '0px',

        // Styling
        background: 'var(--color-white)',
        borderTopLeftRadius:  'var(--radius)',
        borderTopRightRadius: 'var(--radius)',

        // Subtle inner borders using box-shadow
        boxShadow: `
          inset 0   1px 0 0 var(--color-gray-200),
          inset 1px  0 0 0 var(--color-gray-200),
          inset -1px 0 0 0 var(--color-gray-200)
        `,
        marginBottom: '10px',
      }}
    >
      {tabs.map(({ name, href, icon: Icon }) => {
        const active = pathname === href;

        return (
          <Link
            key={href}
            href={href}
            scroll={true}
            className="flex items-center justify-center gap-2 py-2 text-sm w-[115px] md:w-[180px] border-2 transition-none"
            style={{
              // Active tab has visible borders (except bottom)
              borderTop:    active ? '2px solid var(--color-gray-900)' : '2px solid transparent',
              borderLeft:   active ? '2px solid var(--color-gray-900)' : '2px solid transparent',
              borderRight:  active ? '2px solid var(--color-gray-900)' : '2px solid transparent',
              borderBottom: '2px solid transparent',     // Always open bottom

              // Rounded top corners only
              borderRadius: 'var(--radius) var(--radius) 0 0',

              // Active tab extends slightly below to cover container border
              marginBottom: active ? '-2px' : '0',

              // Active tab appears above inactive tabs
              zIndex: active ? 1 : 0,

              // Visual differentiation
              fontWeight: active ? 600 : 500,
              color:      active ? 'var(--color-gray-900)' : 'var(--color-gray-700)',
              background: active ? 'var(--color-white)' : 'transparent',
            }}
          >
            <Icon className="w-4 h-4" strokeWidth={active ? 2.5 : 2} />
            {name}
          </Link>
        );
      })}
    </div>
  );
}
Dependency Required

This component uses lucide-react for icons. Install it with: npm install lucide-react

Key Styling Techniques

Centering at bottom: The tabs are centered using left: 50% combined with transform: translateX(-50%). This is more reliable than flexbox centering for absolutely positioned elements.

Active tab effect: The active tab has a negative marginBottom of -2px which makes it extend slightly below the tab container, visually connecting it to the content area below.

Open bottom border: All tabs have a transparent bottom border, creating the "folder tab" effect where the active tab appears to be part of the content area.

6. Step 4: TabbedContentWrapper

This component combines the Frame and FloatingTabs into a single wrapper that you'll use in your layout.

components/TabbedContentWrapper/index.tsx
'use client';

import React from 'react';
import Frame        from './Frame';
import FloatingTabs from './FloatingTabs';

export default function TabbedContentWrapper({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <Frame>
      {/* Scrollable content area */}
      <main
        className="flex-1 overflow-auto bg-transparent pointer-events-auto hide-scrollbar"
        data-scroll-restoration-id="main-content"
        style={{ borderRadius: 'calc(var(--radius) - 4px)' }}
      >
        {children}
      </main>

      {/* Floating tabs at bottom */}
      <FloatingTabs />
    </Frame>
  );
}

Component Breakdown

  • flex-1 - Makes the main content area fill available vertical space
  • overflow-auto - Enables scrolling when content exceeds viewport
  • hide-scrollbar - Uses our CSS class to hide the scrollbar visually
  • data-scroll-restoration-id - Helps with scroll position restoration on navigation
  • The border radius is slightly smaller than the Frame to account for the Frame's border width

7. Step 5: Layout Integration

Finally, wrap your entire application with the TabbedContentWrapper in your root layout file.

app/layout.tsx
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";
import "./globals.css";

import TabbedContentWrapper from "@/components/TabbedContentWrapper";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "Your App Title",
  description: "Your app description",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body className={`${geistSans.variable} ${geistMono.variable} antialiased`}>

        <TabbedContentWrapper>
          {children}
        </TabbedContentWrapper>

      </body>
    </html>
  );
}
Path Alias

The @/components path alias requires configuration in tsconfig.json. This is set up by default in Next.js projects created with create-next-app.

8. Customization Guide

Adding or Modifying Tabs

Edit the tabs array in FloatingTabs.tsx:

const tabs = [
  { name: 'Home',     href: '/',         icon: Home     },
  { name: 'Products', href: '/products', icon: Package  },
  { name: 'Cart',     href: '/cart',     icon: ShoppingCart },
  { name: 'Account',  href: '/account',  icon: User     },
];

Changing Tab Width

Modify the width classes in the Link component:

// Current: 115px mobile, 180px desktop
className="... w-[115px] md:w-[180px] ..."

// Example: 100px mobile, 150px desktop
className="... w-[100px] md:w-[150px] ..."

Changing Colors

Update the CSS variables in globals.css:

:root {
  --color-gray-900: #111827;  /* Border color for active tab */
  --color-gray-700: #374151;  /* Text color for inactive tabs */
  --color-gray-200: #e5e7eb;  /* Box shadow borders */
}

Changing Frame Width

Modify the max-w-6xl class in Frame.tsx:

// Current: max-width 1152px
className="... max-w-6xl ..."

// Options:
// max-w-4xl = 896px
// max-w-5xl = 1024px
// max-w-7xl = 1280px
// max-w-full = 100%

Removing the Frame Border

For a more minimal look, remove the border in Frame.tsx:

style={{
  margin: '8px',
  // border: '1px solid var(--color-gray-900)',  // Remove this line
  borderRadius: 'var(--radius)',
  background: 'var(--color-white)',
}}

9. How It Works

Active Tab Detection

The component uses Next.js's usePathname() hook to get the current URL path. It compares this against each tab's href to determine which tab should appear active:

const pathname = usePathname();  // e.g., "/portfolio"
const active = pathname === href;   // true if tab matches current route

Route Prefetching

To ensure instant navigation, routes are prefetched when the component mounts:

useEffect(() => {
  router.prefetch('/');
  router.prefetch('/portfolio');
}, [router]);

Pointer Events Strategy

The implementation uses a clever pointer-events strategy to allow background elements to receive clicks while still having a clickable content area:

  1. Outer Frame div: pointer-events-none (clicks pass through)
  2. Inner content div: pointer-events-auto (clicks work normally)
  3. Tabs: pointer-events-auto (clicks work normally)

This allows the background of the page (outside the Frame) to potentially be interactive if needed.

The "Open Bottom" Tab Effect

The classic folder-tab effect (where the active tab connects to the content below) is achieved through:

  1. Active tab has a transparent bottom border
  2. Active tab has marginBottom: -2px which extends it slightly down
  3. Active tab has zIndex: 1 to appear above other tabs
  4. Active tab has the same background as the content area
